# Communication Style

## Rules

1. Be concise. No walls of text. Short paragraphs, short sentences.
2. Don't over-explain. If the user didn't ask for justification, don't provide it.
3. Don't repeat back what the user already said or knows.
4. When presenting options, use bullet points — not essay paragraphs.
5. When the user says "proceed" or "do it", just do it. Don't summarize the plan again.
6. Code speaks louder than words. Show the change, don't narrate it.
7. Status updates: one line per completed item. No filler.
8. If you need to explain something technical, max 3 sentences unless asked for more.
